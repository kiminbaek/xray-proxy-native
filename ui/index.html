用法（SSH 连到 NAS）：

# 临时把单条命令走代理
/vol3/@appcenter/xray-proxy-native/bin/with-proxy curl https://google.com
/vol3/@appcenter/xray-proxy-native/bin/with-proxy apt update
/vol3/@appcenter/xray-proxy-native/bin/with-proxy git clone https://github.com/xxx
/vol3/@appcenter/xray-proxy-native/bin/with-proxy pip install xxx

# 一次性给整个 shell 走代理（之后所有命令都走代理）
export PATH=/vol3/@appcenter/xray-proxy-native/bin:$PATH
source /vol3/@appcenter/xray-proxy-native/bin/enable-proxy.sh
curl https://google.com
apt update

支持的应用：apt / curl / wget / git / pip / npm / qBittorrent / Aria2 / Transmission 等所有尊重 LD_PRELOAD 的 C/C++ 程序。

不支持：docker 守护进程 / 大部分飞牛官方应用（Go 写的，忽略 LD_PRELOAD）/ Chromium / Firefox。

Docker 镜像拉取：用应用中心自带的代理设置（指向 192.168.2.132:10809），不动系统。

原理：LD_PRELOAD 注入 libproxychains4.so（48KB），劫持 connect() 把流量转给 xray SOCKS5（127.0.0.1:10808）。

回滚：停用本应用 → with-proxy 命令变空壳（连不上 10808）→ 网络 100% 恢复。
