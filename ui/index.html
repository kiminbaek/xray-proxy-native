用法（SSH 连到 NAS）：

# 临时把单条命令走代理
/vol3/@appcenter/xray-proxy-native/bin/with-proxy curl https://google.com
/vol3/@appcenter/xray-proxy-native/bin/with-proxy apt update
/vol3/@appcenter/xray-proxy-native/bin/with-proxy git clone https://github.com/xxx
/vol3/@appcenter/xray-proxy-native/bin/with-proxy pip install xxx

# 一次性给整个 shell 走代理（之后所有命令都走代理）
export PATH=/vol3/@appcenter/xray-proxy-native/bin:$PATH
source /vol3/@appcenter/xray-proxy-native/bin/enable-proxy.sh
curl https://google.com
apt update

支持的应用：apt / curl / wget / git / pip / npm / qBittorrent / Aria2 / Transmission 等所有尊重 LD_PRELOAD 的 C/C++ 程序。

不支持：docker 守护进程 / 大部分飞牛官方应用（Go 写的，忽略 LD_PRELOAD）/ Chromium / Firefox。

Docker 镜像拉取：用应用中心自带的代理设置（指向 192.168.2.132:10809），不动系统。

原理：LD_PRELOAD 注入 libproxychains4.so（48KB），劫持 connect() 把流量转给 xray SOCKS5（127.0.0.1:10808）。

回滚：停用本应用 → with-proxy 命令变空壳（连不上 10808）→ 网络 100% 恢复。

⚠️ 高风险操作 · 动路由表的安全提示
TUN 模式会修改系统路由表 + iptables 规则，接管本机全部 TCP/UDP 流量。
多重保护：
① 卸载时自动清理（cmd/uninstall_callback 删 tun 设备 + 清路由表 + 清 iptables）
② 升级时自动清理（cmd/upgrade_callback 先清旧残留再装）
③ xray 崩溃时自动清理（守护进程检测到 + TUN 启用中 → 立即清理）
④ 手动恢复脚本：sudo bash /vol3/@appdata/xray-proxy-native/backup_network.sh
如不需要"全局代理"，强烈推荐用 Socks 模式（不动路由表，更安全）。
什么是 TUN 模式？
默认代理（10808 SOCKS5 / 10809 HTTP）需要客户端主动配置代理；TUN 模式让本机所有进程的流量都自动走 xray —— 透明代理，无需任何客户端配置。

工作原理：xray-core 26.6.1 创建虚拟网卡 tun0，autoSystemRoutingTable 自动接管 default 路由，autoOutboundsInterface 指定出站网卡（留空自动检测）。
安全机制：二进制已 setcap 提权（cap_net_admin + cap_net_raw），daemon 跑 root（需 mihomo 同款 privilege 方案），安装时自动备份原始路由。
前置条件：fpk 内置 config/privilege=root（mihomo 同款通用方案），setcap 失败 TUN 不可用。升级会自动重跑 setcap。
默认 tun0，留空使用默认值
TUN 设备网关（数组形式），避免与局域网冲突
TUN 设备 DNS（数组形式）
如不知道网卡名，先 SSH 执行 ip route | grep default 查看
