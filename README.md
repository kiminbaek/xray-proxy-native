# Xray 代理（xray-proxy-native）

面向飞牛 fnOS 的 Xray 代理桌面工具，提供 SOCKS/HTTP 代理、节点管理、测速、流量统计、日志、备份恢复，以及可选的 TUN 透明代理能力。

> 默认推荐使用 SOCKS/HTTP 代理；TUN 透明代理属于高级功能，默认手动启用，不随应用启动自动恢复。

---

## 功能概览

### 基础代理能力

- 内置 Xray Core，安装后即可运行管理面板。
- 支持 SOCKS5 代理入口。
- 支持 HTTP 代理入口。
- 支持节点导入、添加、编辑、删除、启用/禁用。
- 支持节点测速与批量测速。
- 支持运行日志查看。
- 支持配置备份、导入和恢复默认配置。

### 节点与稳定性

- 节点健康检查。
- 节点延迟历史。
- 自动重启守护。
- 自动选择最优节点配置界面。
- 可手动立即切换到当前最优节点。

### 流量统计

- 展示总下载/上传流量。
- 展示 inbound 实际消费流量。
- 展示 outbound 节点流量排行。
- 支持 24 小时流量历史。

从 v1.20.0 起，总流量优先使用 inbound 统计，即 SOCKS/HTTP/TUN 实际消费流量，避免页面长期显示 0B。

### 通知能力

支持以下通知渠道配置：

- Telegram Bot
- 企业微信 Webhook
- Server 酱

可用于：

- Xray 崩溃提醒
- 自动重启失败提醒
- 节点健康异常提醒
- 登录锁定提醒
- 流量告警

### TUN 透明代理

TUN 模式用于让不支持手动代理配置的应用也能走代理。该模式会接管系统流量并涉及路由/iptables，因此默认不自动启用。

提供：

- TUN 手动启用/停止。
- TUN 配置保存。
- TUN 状态检测。
- TUN 残留清理按钮。
- TUN 只读诊断面板。
- TUN 安全向导。
- 一键导出诊断包。

## 端口说明

| 用途 | 端口 |
|---|---:|
| 管理面板 / fnOS iframe | `2088` |
| SOCKS5 代理 | `10808` |
| HTTP 代理 | `10809` |
| Xray 内部 API | `10085` |

> `10085` 是 Xray 内部 stats/API 使用端口，不是管理面板端口。

## 使用说明

### 1. 安装并打开应用

安装 fpk 后，在 fnOS 应用中心打开 **Xray 代理**。

首次使用需要设置管理密码。

### 2. 导入或添加节点

进入 **节点** 页面：

- 可粘贴分享链接或订阅内容导入；
- 可手动添加 VLESS / VMess / Trojan / Shadowsocks 等节点；
- 可对节点执行测速；
- 可启用、禁用、编辑或删除节点。

### 3. 使用 SOCKS/HTTP 代理

默认推荐使用 SOCKS/HTTP 代理，不修改系统路由，风险最低。

客户端代理配置：

```text
SOCKS5: 127.0.0.1:10808
HTTP:   127.0.0.1:10809
```

如果其他设备要使用 NAS 上的代理，请将地址改为 NAS IP。

### 4. 查看流量

进入 **流量** 页面可查看：

- 总下载/上传；
- inbound 入口流量；
- outbound 节点排行；
- 历史趋势。

### 5. 自动选优

进入 **增强** 页面，可配置自动选择最优节点：

- 是否启用；
- 稳定次数；
- 最小切换间隔；
- 平均窗口；
- 最大延迟；
- 立即切换最优节点。

### 6. 通知配置

进入 **增强** 页面，可配置通知渠道并发送测试通知。

密钥类输入框留空表示不修改已有密钥。

### 7. TUN 诊断与安全向导

进入 **诊断** 页面：

- 查看 TUN 配置状态；
- 检查 `/dev/net/tun`；
- 检查 TUN 设备是否存在；
- 检查 iptables 是否有 `XRAY` 链；
- 检查路由表是否有 TUN 设备路由；
- 导出诊断包。

诊断接口只读，不会启用 TUN，不会修改路由，不会清理残留。

## 安全说明

### SOCKS/HTTP 模式

推荐默认使用该模式：

- 不修改系统路由；
- 不接管全局网络；
- 风险低；
- 适合浏览器、CLI、容器等支持代理配置的场景。

### TUN 模式

TUN 模式会接管网络流量，适合无法手动配置代理的应用，但风险更高。

使用建议：

1. 启用前先查看 TUN 诊断面板。
2. 确认没有 TUN 路由残留。
3. 确认没有异常 `XRAY` iptables 链。
4. 确认网络备份脚本存在。
5. 保持 `autoStart=false`。
6. 出现异常时先停止 TUN，再手动点击“清理残留”。

## 诊断包说明

诊断包接口：

```text
GET /api/tun/diagnostic-package
```

导出的 JSON 包含：

- 应用状态；
- TUN 配置；
- TUN 设备状态；
- `ip addr`；
- `ip route`；
- `ip rule`；
- `iptables -S`；
- 相关进程列表。

诊断包会脱敏：

- token；
- password；
- password_hash；
- 长 UUID/id。

## 常见问题

### 页面流量一直显示 0B

请升级到 v1.24.3 或更新版本。

v1.20.0 已将总流量改为优先使用 inbound 实际消费流量统计。

### 打开网络模式页面显示不全

请升级到 v1.19.3 或更新版本。

v1.19.3 已修复 TUN 面板滚动和按钮被裁切的问题。

### TUN 开启后网络异常怎么办

建议顺序：

1. 先在界面点击“停止 TUN”；
2. 再点击“清理残留”；
3. 如界面无法打开，使用诊断页面提示的恢复脚本；
4. 必要时在 fnOS 应用中心停用/卸载应用后再执行恢复。

### 为什么不默认启用 TUN

因为 TUN 会修改系统网络路径，异常配置可能导致 NAS 网络不可达。应用默认推荐 SOCKS/HTTP，TUN 只作为高级手动功能。
